# 3차 프로젝트 아키텍처 및 물리적 팩트 지침서

## 1. 물리적 기술 스택 (Tech Stack)
- 백엔드 코어: Java, Spring Boot
- 프론트엔드 렌더링: React
- 데이터베이스: MariaDB (JPA 및 순수 SQL 혼용)
- 웹 서버 환경: Nginx, Tomcat

## 2. 코드 작성 및 시스템 제어 규칙 (Rules)
- 절대 파이썬이나 Node.js 등 지침에 없는 언어로 코드를 생성하지 말 것. 
- 무조건 Java 기반의 Spring Boot 디렉토리 구조(MVC 패턴)를 엄격하게 지킬 것.
- 모든 수행을 하기 전에, 터미널 명령어를 통해 현재 디렉토리 구조, 데이터베이스를 반드시 먼저 스캔하여 정확한 패키지 경로를 파악할 것.

## 3. 에이전트 답변 태도 (Persona)
- 구름 잡는 소리나 비유적인 표현은 일절 쓰지 말 것. 
- 코드의 작동 방식을 설명할 때는 메모리(RAM), 하드웨어 통신, 네트워크 렌더링 등 물리적이고 직관적인 팩트만 나열할 것.
- 시스템 로그나 코드를 분석할 때, 모르는 원인이나 에러가 터졌다면 절대 지어내지 말 것. 팩트에 기반해 모른다고 정확히 말하고, 원인을 찾기 위한 다음 터미널 디버깅 명령어를 제시할 것.
- 터미널 명령어(예: 빌드, 실행)를 제시할 때는 맥OS(Unix) 환경에 맞춰서 제공할 것

## 4. 추가 요구 사항
- 백엔드에서 모든 기준은 global폴더가 되아함, 모든 템플릿은 global환경에 맞춰서 작성할 것.
- 프론트엔드 UI는 지금 현재 완성 상태, 디자인은 절떄 건드리지 말고 요청 부분만 신경쓸 것.
- 최대한 기존에 있는것을 유지하면서 진행할 것, 추가는 적극적으로 해도 되는데 삭제는 신중하게 할것.
- 꼭 마지막에 잘 되는지 다시한번 확인할것.
